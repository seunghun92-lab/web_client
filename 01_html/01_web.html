웹(Web)의 역사와 현재: 연결을 넘어 생태계로

1. 웹의 탄생: 정보 공유를 위한 시작

웹(World Wide Web)은 1989년 팀 버너스리 경(Sir Tim Berners-Lee)이 CERN(유럽 입자 물리 연구소)에서 제안했습니다. 원래 목적은 전 세계에 흩어진 연구원들이 논문과 데이터를 쉽게 공유하기 위함이었습니다.

1990년 12월, 팀 버너스리는 세계 최초의 웹 브라우저와 웹 서버를 개발했고, 이 기술을 특허 없이 무료로 공개했습니다. 이 결정은 웹이 특정 기업의 소유물이 아닌, 인류 모두의 공공재로 폭발적인 성장을 할 수 있었던 결정적 계기가 되었습니다.

2. 인터넷과 웹: 도로와 건물의 관계

많은 사람들이 혼용하지만, 기술적으로 인터넷(Internet)과 웹(Web)은 명확히 다릅니다.

  • 인터넷 (도로망): 전 세계 컴퓨터를 물리적으로 연결하는 거대한 통신 네트워크(인프라)입니다.
  • 웹 (건물/서비스): 인터넷이라는 도로 위에 지어진 정보 서비스 중 하나입니다. (이메일, FTP, 온라인 게임 등도 인터넷 도로를 사용하는 다른 건물들입니다.)
요약: 인터넷이 없으면 웹도 존재할 수 없지만, 웹이 없어도 인터넷은 존재합니다.

3. 웹을 구성하는 3대 기술 (The Trinity)

프론트엔드 개발은 다음 세 가지 기술의 협업으로 이루어집니다. 이를 인체에 비유하면 이해가 쉽습니다.

HTML (구조/뼈대)

웹페이지의 뼈대를 만듭니다.

<!-- 이것은 버튼이라는 구조를 정의함 -->
<button>로그인</button>

CSS (표현/피부)

시각적인 디자인을 담당합니다.

/* 버튼을 파란색으로 칠하고, 모서리를 둥글게 만듦 */
button {
  background-color: blue;
  border-radius: 5px;
}

JavaScript (동작/근육)

동적인 기능을 수행합니다.

// 사용자가 버튼을 클릭했을 때의 동작을 정의함
button.addEventListener('click', () => {
  login(); // 로그인 함수 실행
});

4. 웹의 진화: 1.0에서 3.0까지

웹은 데이터의 소유와 상호작용 방식에 따라 세대로 구분됩니다.

세대 시기 별칭 특징
Web 1.0 ~2004 읽기 전용
(Read Only)		 운영자가 정보를 일방적으로 제공합니다.
(신문, 방송과 유사)
Web 2.0 2004~ 읽기/쓰기
(Read-Write)		 사용자가 직접 글과 영상을 올리고 소통합니다.
플랫폼(Google, Meta)이 데이터를 중앙 관리합니다.
Web 3.0 미래/현재 소유
(Read-Write-Own)		 블록체인 등을 통해 개인이 자신의 데이터를
직접 소유하고 관리하는 탈중앙화 웹입니다.

5. 시각화: 웹사이트 vs 모바일 앱 성장 추이

스마트폰의 등장 이후 웹은 모바일 앱(App)이라는 강력한 경쟁자이자 파트너를 만났습니다. 아래 그래프는 2008년부터 2024년까지 두 생태계의 양적 성장을 비교한 것입니다.

그림 1: 웹사이트(파란선)와 앱(빨간선) 성장 비교 (로그 스케일)

그래프 해석 (로그 스케일 적용)

위 차트는 데이터 규모의 차이를 한눈에 보기 위해 로그 스케일(Log Scale)로 작성되었습니다. Y축의 한 칸이 10배의 차이를 의미한다는 점에 유의해야 합니다.

  • 규모의 차이 (Height):
    • 파란선 (웹사이트): 10억(1B) 개 단위에서 움직입니다. 웹의 개방성 덕분에 그 수가 압도적으로 많습니다.
    • 빨간선 (모바일 앱): 100만(1M) 개 단위에서 움직입니다. 스토어 심사와 설치 장벽으로 인해 웹보다 수는 적습니다.
  • 성장의 흐름 (Trend):
    • 폭발적 성장: 2010년대 초반, 스마트폰 보급과 함께 앱 시장(빨간선)이 수직 상승하며 웹을 위협했습니다.
    • 성숙과 안정: 2017년 이후 두 그래프 모두 기울기가 완만해지거나 꺾였습니다. 이는 시장이 양적 팽창에서 질적 성숙 단계로 진입했음을 의미합니다.

결론: 경쟁과 공존

데이터 수(Quantity)는 웹이 압도적이지만, 실제 사용 시간(Time Spent)은 앱이 전체 모바일 사용 시간의 90%를 차지하며 압도적입니다.

  • 웹(Web): 검색을 통한 정보 획득, 설치 없는 접근성 (넓은 연결)
  • 앱(App): 자주 쓰는 서비스의 높은 성능과 몰입감 (깊은 사용)

현대의 개발자는 이 두 플랫폼의 특성을 이해하고, 서비스 목적에 맞는 기술 스택을 선택해야 합니다.
